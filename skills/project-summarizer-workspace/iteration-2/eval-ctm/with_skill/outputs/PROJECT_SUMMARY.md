# Project Summary: ctm

## Identity

- **What it is:** `ctm` ("Claude Tmux Manager") is a single-binary Go CLI that manages [Claude Code](https://docs.anthropic.com/) sessions inside `tmux`, with an embedded HTTP daemon (`ctm serve`) that exposes a React/Vite SPA on `127.0.0.1:37778` for browser/mobile control of those sessions. Per `README.md`: "survive SSH drops, reattach from your phone."
- **Type:** Hybrid — **CLI tool with embedded HTTP daemon and React SPA**. Top archetype is CLI; the daemon side is a non-trivial web service in its own right and is documented separately in `docs/project/architecture.md`.
- **Status:** Active — last commit on `main` is the conversation-start checkpoint at 2026-04-26 (`git log -1`); the previous run of normal feature commits is dense across 2026-04-16..22.
- **Primary language(s):** Go 1.25.0 (`go.mod:3`), TypeScript 5.6 + React 19 (`ui/package.json`).

## Tech stack

- **Backend:** Go 1.25.0, `cobra` v1.10.2 for CLI, `mattn/go-sqlite3` v1.14.42 (CGO-disabled — see Gotchas), `chzyer/readline` v1.5.1, `fsnotify` v1.7.0, `golang.org/x/crypto` v0.50.0. (`go.mod`).
- **Frontend:** React 19.2.5, React Router 7.14.1, TanStack Query 5.99.2, TailwindCSS 4.2.2, Radix UI primitives, `@microsoft/fetch-event-source` for SSE auth (`ui/package.json`). Build tool: Vite 7.0. Component scaffolding: shadcn/ui (`ui/components.json`).
- **Datastore(s):**
  - Two JSON state files in `~/.config/ctm/`: `config.json` and `sessions.json` — versioned, migrated, and flock-protected via `internal/migrate` and `internal/jsonstrict`.
  - SQLite at `~/.config/ctm/ctm.db` — opened via `internal/serve/store/cost_store.go` for per-session cost history and an FTS5-backed `tool_calls_fts` virtual table (V19 search). Wiped and replayed from the JSONL log tailer on every daemon boot.
  - Per-session JSONL tool-call logs at `~/.config/ctm/logs/<claude-session-uuid>.jsonl`, written by the hidden `ctm log-tool-use` subcommand and tailed by `internal/serve/ingest/tailer.go`.
- **Build system:** GNU `make` (`Makefile`) orchestrating `pnpm` (Vite build) and `go build`.
- **Test framework(s):** Go `testing` (unit + race + integration tag), Vitest 3 (UI unit), Playwright 1.59 (UI E2E with mocked `/api`+`/events`), `govulncheck`, `pnpm audit` — all wired through `make regression`.
- **Manifest files read:** `go.mod`, `go.sum`, `ui/package.json`, `ui/vite.config.ts`, `ui/playwright.config.ts`, `ui/vitest.config.ts`, `Makefile`, `.github/workflows/release.yml`.

## Run, build, test

Commands taken from `Makefile`, `ui/package.json`, and `.github/workflows/release.yml`.

```bash
# First-time setup (UI deps required even just to `go build` because of the embed)
make ui                  # cd ui && pnpm install --frozen-lockfile && pnpm build && rsync ui/dist/ internal/serve/dist/

# Build binary (requires UI to be built first; tags are mandatory — see Gotchas)
make build               # make ui && go build -trimpath -tags sqlite_fts5 ./...

# Local development loop (parallel: pnpm dev + go run . serve)
make dev                 # vite on :5173 proxies /api,/events,/healthz to :37778; SIGINT tears both down

# Playwright E2E (requires `pnpm exec playwright install chromium` once)
make e2e                 # pnpm build then `pnpm exec playwright test` against vite preview

# Pre-merge regression pack (~60-90 s on warm Go cache; fails fast)
make regression          # go build/test/race ./internal/serve/... + govulncheck + pnpm tsc --noEmit + pnpm test + pnpm audit --audit-level=high + make e2e

# Integration tests (require a built ./ctm binary)
go test -tags integration ./...   # see integration_test.go

# Single test
go test -tags sqlite_fts5 ./internal/tmux/ -run TestSendKeys -v
```

## Conventions an agent must respect

The top rules; full set in `docs/project/conventions.md`.

1. **Every `go build`, `go test`, `go install` MUST pass `-tags sqlite_fts5`.** Without it, `mattn/go-sqlite3` is compiled without the FTS5 module and the daemon panics at boot with `"no such module: fts5"`. The `Makefile` wires this into every Go invocation; ad-hoc commands must do the same. (`Makefile:11-15`).
2. **CLI dispatcher style is hand-rolled-on-cobra:** one verb per file under `cmd/<verb>.go`, each with a top-level `var <verb>Cmd = &cobra.Command{…}` and an `init()` that registers it with `rootCmd`. **Don't refactor into a `commands/` slice or generator.** Adding a new command means: new file in `cmd/`, mirror the structure, write a sibling `*_test.go`. (`cmd/serve.go`, `cmd/yolo.go`, `cmd/overlay.go` are good templates.)
3. **HTTP routes live entirely in `internal/serve/server.go:registerRoutes`** as `mux.Handle("METHOD /path", authHF(api.Handler(...)))` lines (Go 1.22 ServeMux path patterns). Handlers themselves are factory functions in `internal/serve/api/<resource>.go` that close over their dependencies (no global state). **Don't introduce a routing framework.**
4. **Auth is mandatory on every JSON endpoint except `/healthz` and `/api/auth/status`.** Mutation routes additionally pass through `api.RequireOriginFunc(allowedOrigins, …)` for CSRF protection. (`internal/serve/server.go:registerRoutes`).
5. **State files (`config.json`, `sessions.json`) are versioned + flock-locked + atomically written**, with `<path>.bak.<unix-nano>` snapshots before any destructive migration. Never `os.WriteFile` these directly — go through `internal/session/Store` or `internal/config.write` and wire migrations through `internal/config.MigrationPlan` / sessions-equivalent. (`internal/migrate/migrate.go` package doc.)
6. **JSON config decoding is strict:** unknown top-level keys cause a one-shot self-heal (rename to `.bak.unknowns.<unix-nano>`, strip, log WARN) on first load, then are rejected from then on. Don't add typed fields without also updating `Default()` and the schema version if it's non-additive. (`internal/config/config.go:Load`, `internal/jsonstrict/`.)
7. **Hooks must never block the action that triggered them.** `cmd/hooks_dispatch.go:fireHook` swallows errors; new lifecycle events should follow the same pattern.
8. **The daemon process never attaches to tmux.** `internal/session/spawn.go:Yolo` deliberately omits attach so the same code path works from CLI (which then attaches) and from the HTTP `POST /api/sessions` endpoint (which doesn't).
9. **UI auth uses a single localStorage bearer token** at `ctm.token`. All fetch goes through `ui/src/lib/api.ts:api()`, which throws `UnauthorizedError` on 401 so `AuthProvider` can clear and redirect.
10. **All Vite HMR + dev runs through the proxy in `ui/vite.config.ts`** — `/api`, `/events`, `/healthz`, `/health` all forward to `127.0.0.1:37778`. SSE streams forward unbuffered (`ws: false` is intentional — SSE is HTTP/1.1 keep-alive, not websockets).

## Gotchas

- **`-tags sqlite_fts5` or boot panic.** See Conventions #1. The single most common way to break this codebase silently is to run `go test ./...` without the tag — most packages pass, then `internal/serve/store/cost_store.go` panics at runtime when SQLite tries to load FTS5. (`Makefile:11-15`, `internal/serve/store/cost_store.go:133-138`.)
- **The UI must be built before `go build`.** `internal/serve/assets.go` declares `//go:embed all:dist` against `internal/serve/dist/`. If that directory is empty (e.g. fresh clone, no `make ui`), the embed succeeds with an empty FS and the daemon serves no SPA. The directory exists in the tree because `go:embed` rejects parent-relative paths — that's why `Makefile` rsyncs `ui/dist/` → `internal/serve/dist/` rather than embedding `../../ui/dist`. (`Makefile` comments lines 8-10.)
- **`make ui` calls `pnpm install --frozen-lockfile`** — first-time runs without `node_modules/` will pull packages into `ui/node_modules/`. Air-gapped installs need a populated pnpm store first.
- **`ctm` itself auto-spawns `ctm serve`.** `internal/serve/proc.EnsureServeRunning` is called from `runAttach`, `runYolo`, `runYoloBang`, `runSafe`. If a non-ctm-serve process already owns `:37778`, the spawn returns silently (single-instance guard via `X-Ctm-Serve` header probe in `internal/serve/server.go:probeIsCtmServe`). Tests that expect a clean port should kill stray daemons first.
- **`yolo` mode does `git checkpoint` automatically** if `cfg.GitCheckpointBeforeYolo` is true. The empty-commit message is `checkpoint: pre-yolo <ISO-timestamp>`. Rollback: `git -C <workdir> reset --hard HEAD~1`. Skipped silently for non-git workdirs. (`cmd/yolo.go:gitCheckpoint`.)
- **The `BuildCommand` `||` fallback fires on ANY non-zero exit from `claude --resume`** — auth error, crash, Ctrl-C all promote to a fresh `--session-id` invocation. Intentional. (`internal/claude/command.go:BuildCommand` doc comment.)
- **`shouldResumeExisting` ignores tmux liveness on purpose.** A regression once made `ctm yolo <name>` reset history when claude had already exited. Resume is now driven only by stored mode equality. Don't re-add a tmux-alive precondition. (`cmd/yolo.go:shouldResumeExisting`.)
- **`ctm` binary at repo root is committed (~13 MB).** It is a build artifact, gitignored in `.gitignore` line `Built binary` — but a stale one is checked in. Don't trust its version; rebuild before testing. `[inferred]` from `ls -la` showing `-rwxrwxr-x ... 13959746 Apr 21 03:23 ctm` and `.gitignore` listing it.
- **Vite dev server is hardcoded to port 5173** (`strictPort: true`); the daemon to 37778. Not overridable via env in code. Both are documented for users in `README.md`.
- **`internal/serve/dist/` is rsynced with `--delete`** on every `make ui`. Anything you put there by hand will vanish.
- **Two parallel-agent worktree dirs leak into greps:** `/home/dev/projects/ctm/.claude/worktrees/agent-a453189e/`, `agent-a686b646`, `agent-a6e183b8` — they are full source-tree copies. Tooling that recurses into the repo root may double-count. Filter `--exclude .claude` or `--exclude worktrees`.
- **Logs at `~/.config/ctm/logs/*.jsonl`** are size+age+count rotated by `internal/logrotate.Prune`, called from `cmd/bootstrap.go:pruneSessionLogs` on every claude-launching command. `[inferred]` that this also runs from `serve` boot — verify via `grep -n pruneSessionLogs internal/serve/`.
