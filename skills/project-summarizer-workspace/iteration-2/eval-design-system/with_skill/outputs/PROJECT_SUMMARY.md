## Conventions an agent must respect

The top rules for safely modifying this code. Full set in `docs/project/conventions.md`.

- **Component prop types live in ONE file** — `src/components.d.ts` holds every component's props interface. This is intentional; do **not** colocate per-component. See `docs/project/conventions.md` "Don't refactor".
- **One file per component family**, not per component — e.g. `feedback.tsx` ships Alert, Modal, Drawer, Progress, Skeleton, Spin, Tooltip, toast, ToastRegion together. Re-exported by family in `src/index.tsx`.
- **CSS class names use `.rcs-*` BEM-ish convention** — see `src/styles.css`. Variants encoded as `.rcs-button--primary`, slots as `.rcs-modal-header`. Don't switch styling solutions.
- **Tokens are CSS variables** in `colors_and_type.css` (root file, NOT inside `src/`). Theme swap via `[data-theme="light|dark"]` on `<html>`. Don't hard-code hex except in the `BRAND_HEX` map in `src/components/theme.tsx`.
- **Adding a new component** = create file in `src/components/`, add prop interface to `src/components.d.ts`, add styles to `src/styles.css` with `.rcs-*` names, add one re-export line to `src/index.tsx`. (See `README.md > Editing components`.)
- **Strict event signatures** — props use typed handlers (`(e: MouseEvent<HTMLButtonElement>) => void`), never `(value: any) => void`. See `src/components.d.ts:39` and the file's header comment.
- **`readonly` on prop fields** including arrays — see the top-of-file comment in `src/components.d.ts`.
- **Zero runtime deps** — only React/React-DOM (peer). Do not introduce a runtime dependency without explicit approval.

## Gotchas

- **No tests exist yet** despite vitest being installed; CI's test step has `continue-on-error: true` (`.github/workflows/ci.yml:40`). Don't trust a green CI as evidence of correctness; manual verification via `preview/*.html` cards is the current QA mechanism.
- **No eslint config** yet; CI lint step also `continue-on-error` (`.github/workflows/ci.yml:36`). Adding an eslint config will start failing CI unless those `continue-on-error` flags are removed in the same change.
- **Build script is a one-liner with embedded Node `-e`** (`package.json:38`) that reads `colors_and_type.css` (root) + `src/styles.css` and writes `dist/styles.css`. If you rename either CSS file, this breaks silently — `tsc` would still succeed but `dist/styles.css` would either error or be incomplete.
- **`tsconfig.build.json` excludes `src/styles.css`** (`tsconfig.build.json:7`) — TypeScript wouldn't include it anyway, but the explicit exclude is a load-bearing reminder that styles ship via the build script's CSS-concat step, not via tsc.
- **Release workflow refuses to publish on tag/version mismatch** — `.github/workflows/release.yml` "Verify tag matches package version" step exits 1 if `git tag` (`vX.Y.Z`) != `package.json:version`. Use `pnpm version <patch|minor|major>` which bumps both atomically.
- **Dual-registry publish** — same package goes to npm (`registry.npmjs.org`) AND GitHub Packages (`npm.pkg.github.com`) under the same `@ossrandom` scope. The GPR job rewrites `publishConfig.registry` in-place via inline Node before `npm publish`. Bumping or changing `publishConfig` in `package.json` may collide.
- **Toast requires consumer to mount `<ToastRegion />`** somewhere in the tree, then call `toast(...)` imperatively. Forgetting the region means `toast()` is silently a no-op. Source: `src/components/feedback.tsx`, exported in `src/index.tsx`.
- **`useTheme` outside `ThemeProvider` returns a no-op fallback** that reads `prefers-color-scheme` and ignores `setMode`/`toggle` calls (`src/components/theme.tsx:65`). Bugs reported as "theme toggle does nothing" usually mean missing provider — not a bug in the toggle.
- **Root `SKILL.md` is shipped CONTENT, not meta-docs** — it describes brand voice/visual rules that consumers of this design system must follow. It is NOT a description of how to work on this repo. Don't treat it as authoritative for build/test/structure questions; cite `README.md` for that.
- **`ui_kits/*/index.html` load React + Babel + lucide via unpkg CDN** (`ui_kits/marketing/index.html:7-9`). These are reference layouts only — they do not gate publish, are not part of the build, and won't run offline. Do not introduce real bundling here without scoping the change explicitly.
- **`preview/*.html` cards target a 700px design width** with custom responsive scaling rules in `preview/_card.css`. Touch targets are pinned to ≥36px height; scaled padding via `clamp()`. Read `preview/_card.css` header comment before changing card layouts.
