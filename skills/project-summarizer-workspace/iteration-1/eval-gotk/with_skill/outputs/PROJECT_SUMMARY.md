## Identity

- **What it is:** A zero-dependency Go CLI that wraps verbose shell tools (`go test`, `git`, `grep`, `tsc`, `eslint`, `ls`, `tree`, `find`, `docker`, `kubectl`, `cargo`, `npm`, `curl`) and reshapes their stdout into compact summaries before the output reaches an LLM's context window. Includes a smart file reader (`gotk read`) and per-invocation analytics (`gotk gain`).
- **Type:** CLI tool (single binary, stdlib-only Go).
- **Status:** v0.1.0 (`main.go:9`). Repo is a fresh tree with no commits yet — `git status` shows all 29 source/doc files as untracked on branch `main`. Last file mtimes are 2026-04-22.
- **Primary language:** Go 1.22 (`go.mod:3`), but built and verified on Go 1.26.2 in this sandbox (`go build` succeeds, `go test ./...` → `ok gotk 0.008s`).

## Tech stack

- **Frameworks / key libs:** none — `go.mod` has zero `require` lines (verified: `go.mod` is 21 bytes total, just `module gotk` + `go 1.22`). Everything is stdlib.

## Entry points

| Entrypoint | File | Purpose |
|------------|------|---------|
| Binary `gotk` | `main.go` | Single CLI entry. Reads `os.Args[1]`, looks it up in the `handlers` dispatch table, falls through to `passthrough()` for unknown commands. |
| Sub-command handlers | `filter_*.go` (one per command family) | Each exports a `func runX(args []string) int` that runs the underlying tool, reshapes stdout, and returns the exit code. |
| `gotk gain` | `gain.go:13 runGain` | Reads `~/.gotk/history.jsonl` (JSONL written by `track.go:appendEvent`) and prints a savings summary. |

The `handler` type is `func(args []string) int` (`main.go:11`); every filter conforms to it.

## Conventions an agent must respect

(Top rules; full set in `docs/project/conventions.md`.)

1. **Zero third-party dependencies.** `go.mod` must stay empty of `require` directives. The whole point of the project is "stdlib only, builds offline" (`README.md` § Install). Don't introduce libraries.
2. **One package, flat layout.** Every `.go` file is `package main` at the repo root. Don't introduce subpackages without a strong reason.
3. **Filter signature is fixed.** Every command handler is `func runX(args []string) int`, registered in the `handlers` map in `main.go:13`. New commands plug into that map.
4. **Stderr and exit codes are preserved verbatim.** Filters reshape stdout only — `cmd.Stderr = os.Stderr` everywhere; exit code comes from `common.go:execExitCode`. See `README.md` § Use: "Exit codes and stderr are preserved. Only stdout is reshaped."
5. **Track raw vs. filtered bytes.** Filters that reshape output should wrap their reader in `countReader` and writer in `countWriter` (`track.go`) so `gotk gain` can compute savings. Pure passthroughs and line-cap passthroughs already get this via the `execWithLineCap*` helpers.
6. **Tests are colocated.** `xxx.go` ↔ `xxx_test.go` in the repo root. Use the existing pattern: pure parsing functions are unit-tested directly without spawning subprocesses (see `filter_gotest_test.go:TestParseGoTestOutput_Failure`).
7. **Pass through interactive / format-changing invocations.** Watch modes, JSON output flags, `--help`, `--version`, and user-supplied `--format` should bypass reshaping (e.g. `filter_gotest.go:isJSONFlag`, `filter_tsc.go` early `--watch`/`--help` check, `filter_lint.go:runLint` early `--format` check).

## Gotchas

- **No git history yet.** `git log` errors with "your current branch 'main' does not have any commits yet". Everything in the tree is untracked on a fresh `main` branch — there is no history to mine for ADRs or rationale.
- **Pre-built binary in `bin/`** (`bin/gotk`, ~32K-ish, mtime 2026-04-22). It is gitignored (`/bin/` in `.gitignore`) and not part of the source. Don't trust it as authoritative; rebuild when in doubt (`make build` or `go install .`).
- **History file silent failures.** `track.appendEvent` swallows every error from `MkdirAll`/`OpenFile`/`Marshal`/`Write` (`track.go:60-78`). Analytics are best-effort by design — never let them break a command. Don't add a `return err` here without rethinking the contract.
- **Two install paths in the README disagree on Makefile.** `README.md` § Install says "no Makefile required" (`go install .`); `GOTK.md` § Install says the same. The `Makefile` is purely optional convenience. New contributors sometimes assume `make build` is the canonical path — it isn't.
- **`isJSONFlag` and `--format` checks are flag-position blind.** `filter_gotest.go:isJSONFlag` matches `-json`, `--json`, `-test.v=json`, and `-json=…`/`--json=…` prefixes anywhere in args. `filter_lint.go:runLint` uses `hasFlag(args, "--format", "-f")` similarly. If a filter is treating a JSON-mode invocation as a normal one (or vice versa), the flag-detection logic is the first place to look.
- **`gotk read` `--max-chars` defaults to 200.** Long lines are truncated at 200 chars by default (`filter_read.go:13 readDefaultMaxChars = 200`). Use `--raw` or `--max-chars 0` (the code likely handles 0; verify in `filter_read.go:renderReadFile` if needed) for full lines.
- **`gotk lint` shells out to `eslint` directly** — no path resolution against `node_modules/.bin`. The user must have a global or shimmed `eslint`. `[inferred]` from `filter_lint.go:exec.Command("eslint", ...)`.
- **Two assistant instruction packs ship with the repo.** `GOTK.md` is the generic markdown pack (referenced via `@GOTK.md` from CLAUDE.md / AGENTS.md). `.github/copilot-instructions.md` is the GitHub Copilot repo-scoped variant. They overlap heavily — keep them in sync when changing rewrite rules.
