## Conventions an agent must respect

The top rules. Full set in `docs/project/conventions.md`.

1. **CSS class naming = `.rcs-*`** (`src/styles.css`). Variants use BEM-ish double-dash: `.rcs-button--primary`, `.rcs-alert--danger`. Always compose with `cx(...)` from `src/internal/cx.ts` ‚Äî don't hand-concatenate strings.
2. **Tokens come from CSS custom properties**, never hardcoded values. Use `var(--accent)`, `var(--bg-0)`, `var(--font-sans)`, etc. ‚Äî full set in `colors_and_type.css`. Hex literals only for the JS-side `BRAND_HEX` map in `src/components/theme.tsx:13`.
3. **Prop interfaces live in `src/components.d.ts`**, not next to components. All extend `BaseProps` (`id`, `className`, `style`, `data-testid`). Arrays are `readonly`. Event handlers use specific React types ‚Äî never `(value: any) => void` (see header comment at top of `components.d.ts`).
4. **Adding a component:** create `src/components/<name>.tsx`, define `<Name>Props` in `src/components.d.ts`, add styles to `src/styles.css` using `.rcs-<name>*` classes, add a re-export line to `src/index.tsx`. (README section "Editing components".)
5. **Theme switching is global side-effectful.** `ThemeProvider` (`src/components/theme.tsx:32`) sets `document.documentElement.setAttribute("data-theme", mode)` in `useEffect`. All CSS variable swaps key off this. No `<html data-theme>` = light mode by default.
6. **`toast` is a module-level singleton** (`src/components/feedback.tsx:226`). It's a plain array + `Set<Listener>`, not a React context. Mount `<ToastRegion />` exactly once near the root; calls from anywhere will surface there.
7. **Strict TS everywhere** ‚Äî `strict`, `noImplicitOverride`, `noFallthroughCasesInSwitch`, `isolatedModules` (`tsconfig.json`). `tsconfig.build.json` excludes tests and `preview/`.
8. **Pure side-effect declaration:** `package.json:29` declares `"sideEffects": ["**/*.css"]` ‚Äî bundlers may tree-shake JS but must keep CSS imports.
