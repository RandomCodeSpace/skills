# Project Summary: ctm

> Generated by `project-summarizer` on 2026-04-26. Audience: AI agents (and humans) who need to understand and modify this codebase. Every claim should be checkable; items marked `[inferred]` were not directly verified.

## Identity

- **What it is:** Claude Tmux Manager — a Go CLI that wraps `claude` (Anthropic's Claude Code CLI) inside named tmux sessions so they survive SSH drops, plus an embedded `ctm serve` HTTP daemon (port 37778) that powers a React/Vite web UI for monitoring and steering those sessions from a phone.
- **Type:** Hybrid — primarily a Go CLI tool that also ships a single-user web app (Go HTTP server + embedded React SPA). Single binary, no external services.
- **Status:** Active — last commit 2026-04-26 (`README.md`, `cmd serve.go`, `internal/serve/server.go` all touched today). Module path `github.com/RandomCodeSpace/ctm`. Latest released format is `v0.1.0` style tags (see release workflow). Pre-1.0.
- **Primary language(s):** Go 1.25.0 (`go.mod:3`), TypeScript 5.6 / React 19 (`ui/package.json`).

## Tech stack

- **CLI framework:** `github.com/spf13/cobra` v1.10.2 (`go.mod:7`). Root command + ~30 subcommands wired in `cmd/`.
- **Storage:**
  - Plain JSON files under `~/.config/ctm/` for state — `config.json`, `sessions.json`, `user.json` (atomic writes via tmpfile + rename, advisory `flock`, `schema_version` + startup migrations). Paths in `internal/config/config.go:215-250`.
  - **SQLite** (`mattn/go-sqlite3` v1.14.42, `go.mod:9`) for the cost-history store at `~/.config/ctm/ctm.db` (`internal/serve/server.go` — `store.OpenCostStore`). Requires the `sqlite_fts5` build tag (FTS5 used for log search; absence panics at boot — see Gotchas).
- **HTTP server:** stdlib `net/http`, plus `github.com/fsnotify/fsnotify` v1.7.0 for log/state file watching, `github.com/chzyer/readline` v1.5.1 for the prompt UI, `golang.org/x/crypto` v0.50.0 (likely Argon2 for `internal/serve/auth/password.go` — `[inferred]`).
- **UI framework:** React 19.2 + `react-router` 7 (`ui/package.json`). Build with Vite 7 + `@tailwindcss/vite` 4. Server state via `@tanstack/react-query` 5.99. shadcn-style primitives in `ui/src/components/ui/` (config in `ui/components.json`). SSE via `@microsoft/fetch-event-source` 2.0.
- **Build system:** GNU `make` (`Makefile`, ~70 lines) orchestrates `pnpm` UI build → `rsync` → Go build with `-tags sqlite_fts5`. UI bundle is then `//go:embed`-ed into the Go binary (`internal/serve/dist/`, see `internal/serve/assets.go`).
- **Test framework(s):** `go test` (with `-race` on serve subtree); `vitest` 3.2 for UI unit (`ui/vitest.config.ts`); `@playwright/test` 1.59 for UI E2E against a `vite preview` build with mocked `/api` + `/events` (`ui/playwright.config.ts`); root `integration_test.go` gated by `//go:build integration`.
- **Manifest files read:** `go.mod`, `go.sum`, `Makefile`, `ui/package.json`, `ui/tsconfig*.json`, `ui/vite.config.ts`, `ui/vitest.config.ts`, `ui/playwright.config.ts`, `.github/workflows/release.yml`.

## Entry points

| Entrypoint | File | Purpose |
|------------|------|---------|
| CLI binary | `main.go` | Two-line shim: `cmd.Execute()` |
| Cobra root | `cmd/root.go` | Defines `rootCmd`, `--verbose` / `--log-level`; configures slog before any subcommand runs |

| UI app | `ui/src/main.tsx` → `ui/src/App.tsx` | React SPA root; mounts `<RouterProvider>` after `ThemeProvider`/`AuthProvider`/`SseProvider`/`AuthGate` |

| Integration tests | `integration_test.go` | Build-tagged `integration`; spawns the compiled `./ctm` binary |

## Conventions an agent must respect

The top set; full list in `docs/project/conventions.md`.

- **One file per Cobra subcommand** under `cmd/`, with `init()` calling `rootCmd.AddCommand(...)` (e.g. `cmd/serve.go:18`). Tests are colocated as `*_test.go`. Match this layout when adding a command.
- **All Go builds, tests, and `go install` invocations require `-tags sqlite_fts5`.** Without it, anything that opens an FTS5 table (V19 search) panics with `no such module: fts5` at boot (`Makefile` header comment).
- **State files are JSON with `schema_version` + atomic writes.** Use the helpers in `internal/jsonstrict` (DisallowUnknownFields) and `internal/migrate` (startup migrations + `.bak.<unix-nano>` snapshots). Never write directly with `json.Encoder`. Reference: `internal/config/config.go`, `README.md` "State file versioning".
- **CLI lifecycle events fan out twice** via `cmd/hooks_dispatch.go`: `fireHook(event, sess)` runs user shell hooks from `config.Hooks[event]`; `fireServeEvent(event, sess)` POSTs to the local daemon's `/api/hooks/:event`. Both are best-effort (failures swallowed). Add both calls when adding a new lifecycle moment. Known events: `on_new`, `on_attach`, `on_kill`, `on_yolo`, `on_safe`; daemon-side: `session_new`, `session_attached`, `session_killed`, `on_yolo`.
- **Diagnostics go to stderr (slog), user output to stdout (`internal/output.Printer`).** Mixing the two breaks scripts piping ctm output (`internal/logging/logging.go` package doc).
- **UI lives behind shared providers** in `ui/src/App.tsx`: `ThemeProvider → QueryClientProvider → AuthProvider → SseProvider → AuthGate`. New routes should use `react-router` v7 declarative routes inside the existing `createBrowserRouter`. Server state must go through React Query; never `useState`+`fetch` for API data.
- **UI alias is `@` → `ui/src/` only** (`ui/vite.config.ts`, `ui/tsconfig.app.json` paths). Don't introduce new aliases.
- **Embed-path constraint:** `//go:embed` rejects parent-relative paths, so the UI bundle is rsync'ed into `internal/serve/dist/` (sibling of the Go file using `embed.FS`). Don't try to embed `ui/dist/` directly. (`Makefile` comment.)

## Gotchas

- **FTS5 build tag.** Every `go build` / `go test` / `go install` must include `-tags sqlite_fts5`. Missing it causes a runtime panic at boot, not a build error. (`Makefile` line ~12, `internal/serve/server.go` opens an FTS5-backed cost/search store.)
- **`ctm serve` single-instance is silent.** If port 37778 is already owned by another `ctm serve` (detected via the `X-Ctm-Serve` header on `/healthz`), the new process exits with status 0. If a *non*-ctm-serve process owns it, it errors out without disturbing the other listener (`cmd/serve.go`, `internal/serve/server.go:probeIsCtmServe`). Don't conclude "it's running" from a zero exit code; hit `/healthz` to be sure.
- **`ctm serve` is auto-spawned** by `attach`/`new`/`yolo` via `proc.EnsureServeRunning(...)` (`cmd/attach.go:55`). Tests and dev loops sometimes need it explicitly stopped — there's no `ctm serve stop`; send SIGTERM to the process or use the in-process `Server.Shutdown(reason)` if you have a handle.
- **Linux-only `/proc` dependence in `internal/claude/process.go`.** `FindClaudeChild` parses `/proc` directly, no `runtime.GOOS` branches; silently fails on macOS/BSD (documented in `docs/robustness-audit.md` row 3). The release workflow still ships darwin binaries — those binaries' `/proc` lookups are no-ops.
- **YOLO mode auto-commits** a git checkpoint before bypassing claude permissions (`cmd/yolo.go:gitCheckpoint`). Plain unsigned commit, no tag/GPG/SHA pin (`docs/robustness-audit.md` row 7). When testing yolo flows in a real repo, expect a new local commit; the audit file says no push/pull.
- **UI E2E never hits a real daemon.** Playwright mocks `/api` + `/events` at the page level (`ui/playwright.config.ts` doc comment, `ui/e2e/`). The webserver is `vite preview --port 4173 --strictPort` against the built `dist/`. `make e2e` rebuilds first; running `playwright test` directly works only if `pnpm build` has run recently. Chromium must be installed via `pnpm exec playwright install chromium` once.
- **`config.HookTimeoutSec = 0` does NOT mean "no timeout" — it means "use default" (5s).** Set a very large number to effectively disable. Same pattern for log rotation caps and attention thresholds (`internal/config/config.go` Resolved() helpers).
- **Vite dev proxy must not buffer SSE.** `/events` is HTTP/1.1 keep-alive (not WS). The proxy config in `ui/vite.config.ts` explicitly sets `ws: false`; don't change it.
- **PRs add tests; the regression pack grows, doesn't get replaced** (`Makefile` regression-target comment). Every fix or feature is expected to land with a unit / vitest / e2e case.
- **`docs/robustness-audit.md`** (dated 2026-04-18) is a known-issues catalogue with file:line references. Read it before claiming any defensive feature is in place — many items are listed as `partial` or `no` (notably: file perms 0644 vs 0600 on backups, schema validation on overlay, signed releases, OpenTelemetry).
- **`integration_test.go` references `ctmRun(t, home, "safe", ...)`** but I did not find a `cmd/safe.go`. The help test asserts `safe` appears in `ctm help` output (`integration_test.go:39`). Either the command is registered indirectly (yolo alias?) or the test is stale — verify before relying on `ctm safe`. `[inferred]`
- **`.claude/worktrees/agent-*` and `.superpowers/brainstorm/...`** appear in the file tree but are agent-runtime artifacts, not source. They contain duplicate copies of `cmd/` files in worktrees — do not edit those.
