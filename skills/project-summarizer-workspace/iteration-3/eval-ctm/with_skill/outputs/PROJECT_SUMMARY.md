- **Key libs:** `spf13/cobra` 1.10.2, `mattn/go-sqlite3` 1.14.42 (FTS5), `chzyer/readline` 1.5.1, `fsnotify/fsnotify` 1.7.0, `golang.org/x/crypto` 0.50.0 (argon2id). See `go.mod`.

- **Datastore:** SQLite (single file) at `~/.config/ctm/cost.db` with FTS5 virtual table `tool_calls_fts`. - **Build system:** GNU Make (`Makefile`) wrapping `go build` + `pnpm build` + `rsync`. UI is `//go:embed`'d via sibling-rsync into `internal/serve/dist/`.
- **Test frameworks:** `go test` (race, FTS5 build tag), Vitest 3.2 (UI unit), Playwright 1.59 (E2E with mocked `/api` + `/events`).

## Conventions an agent must respect

The top rules. Full set in `docs/project/conventions.md`.

1. **Always build with `-tags sqlite_fts5`.** `mattn/go-sqlite3` only compiles FTS5 when this tag is set. Missing it produces a binary that panics at boot with "no such module: fts5". Source: `Makefile` comment block.
2. **UI bundle must be rsync'd into `internal/serve/dist/` before `go build`.** `//go:embed` rejects parent-relative paths, so `ui/dist/` is mirrored sibling-side. Use `make ui` or `make build` ‚Äî never bare `go build`. Source: `Makefile` ui target.
3. **Atomic + flock writes for state files.** All edits to `sessions.json` / `config.json` go through `internal/jsonstrict` and `internal/session/state.go`. Direct writes are forbidden; the migrator copies bytes to `<path>.bak.<unix-nano>` before any destructive write.
4. **Schema versioning is enforced.** `config.json` and `sessions.json` carry `"schema_version"`; a newer-than-known version causes refusal to start (no silent downgrade). Add new versions only via `internal/migrate` `Plan.Steps`.
5. **Mutation endpoints require Origin allowlist + cookie session.** `mux.Handle("POST /api/...", authHF(api.RequireOriginFunc(allowedOrigins, ...)))`. Don't add a mutation without both wrappers. Recipe in `conventions.md`.
6. **Handler files own a single resource family.** `internal/serve/api/<resource>.go` contains the handler closures; route registration stays in `internal/serve/server.go`. Don't register routes inside handler files.
7. **Strict-mode SSE.** SSE responses set no-buffer headers and are HTTP/1.1 keep-alive (not WebSocket). Vite proxy in `ui/vite.config.ts:31‚Äì40` documents the constraint.

## Gotchas

- **Build tag is non-negotiable.** `go build .` without `-tags sqlite_fts5` produces a binary that boots, accepts traffic, then panics on the first FTS5 query. Always `make build`. Same applies to `go test`.
- **`make ui` is a hard prerequisite for `go build`.** First clean clone needs `make ui` once before `go build` will succeed (the embed target dir must exist with content).
- **`checkpoint: pre-yolo ‚Ä¶` commits flood git log.** YOLO mode auto-creates a checkpoint commit before bypassing permissions. These are not feature work ‚Äî filter them out when summarizing recent activity (`git log --invert-grep --grep='^checkpoint'`).
- **Auth model just changed (V27).** Pre-V27 used a paste-token bearer flow; V27 switched to argon2id password + in-memory cookie session with 30-day TTL. The pre-V27 bearer-token loader was removed in `92eebab`. Don't reintroduce it.
- **UI E2E mocks the entire backend.** `ui/e2e/*.spec.ts` mock `/api` + `/events` at page level. They do not exercise the Go daemon. Treat E2E pass as a UI-contract signal, not a full-system signal.
- **`docs/v02/` and `docs/superpowers/specs/`** are the source of truth for *why* a feature is shaped the way it is. Read the relevant spec before changing attention triggers (A‚ÄìG), mutation auth, or the FTS5 search surface ‚Äî see `docs/project/architecture.md` for the index.
- **CLI verb count is large (~30 verbs).** PROJECT_SUMMARY only lists the 5‚Äì7 most-used. Full table in `docs/project/conventions.md`.
- **Single-instance daemon.** `ctm serve` exits status 0 if another `ctm serve` already owns the port (detected via `X-Ctm-Serve` response header on `/healthz`). Don't replace this with a `--force` flag without thinking through the auto-spawn race.
