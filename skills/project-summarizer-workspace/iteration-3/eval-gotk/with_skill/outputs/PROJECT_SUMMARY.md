## Conventions an agent must respect

Top rules — full set in `docs/project/conventions.md`:

1. **Zero third-party deps.** `go.mod` must stay at "module gotk / go 1.22" with no `require` block. Stdlib only. This is the *whole pitch* — see `README.md` "Why this exists" and `GOTK.md` "Install on restricted machines". A new dep is a breaking design change, not a routine PR.
2. **Flat `package main` at repo root.** No `cmd/`, no `internal/`, no `pkg/`. One file per filter (`filter_<tool>.go`). Don't introduce subdirectories.
3. **One filter per `filter_<tool>.go` file** with a `run<Tool>` function and a colocated `filter_<tool>_test.go`. Register in `main.go:13` `handlers` map and add a row to `usageText` (`main.go:31`).
4. **Only stdout is reshaped; stderr and exit codes pass through verbatim.** Verified: every `runX` sets `cmd.Stderr = os.Stderr` and returns `execExitCode(cmd.Wait())`. Don't break this — it's stated in `README.md` "Use" and `copilot-instructions.md` "Exit codes and semantics".
5. **Wrap with `startRun(label) → ctx.finish(exit)`** so `gotk gain` analytics work. `addRaw` for total bytes consumed from the child, `addFiltered` for bytes emitted. See any `runX` for the pattern, e.g. `filter_grep.go:18,46`.
6. **Always strip color** (`--color=never` / `-c color.ui=never`) before piping into a parser. ANSI is stripped via `stripANSI()` (`common.go:8`) where the parser must tolerate dirty input.
7. **No network calls, no telemetry, no auto-update.** Stated as a non-goal in `GOTK.md` "Non-goals for v0.1" and `README.md` "Non-goals". History is local-only.

## Gotchas

- **No commits yet on `main`.** Everything is untracked. Don't assume `git log` will help with archaeology — there is none. Confirm via `git -C /home/dev/projects/gotk log --oneline --all`.
- **No CI.** Confirm via `ls .github/workflows/` (no such directory). If you add automation, you're the first.
- **Filter caps are constants, not flags.** `grepTotalCap = 200`, `dockerLineCap = 300`, etc. — all in package-level `const ()` blocks at the top of each `filter_*.go`. To change, edit the constant.
- **`gotk go test` falls back to passthrough if `-json` is set** (`filter_gotest.go:13` → `isJSONFlag`). Same for `tsc --watch / -w / --help` (`filter_tsc.go:18-22`) and `eslint --format/-f` already specified (`filter_lint.go:60-62`). Filters intentionally don't fight the user.
- **`gotk grep` falls back to plain passthrough** when format-changing flags are set (`-l`, `-L`, `-c`, `-q`, `-o`, `-Z` and their long forms). See `filter_grep.go:101-127` `grepHasFormatChangingFlag`.
- **`gotk git diff` may run `git` twice** — once for the full diff, once for `--stat` when >300 lines (`filter_git.go:78-103`). Side-effect-free, but worth noting if you're profiling.
- **`gotk lint` always runs eslint with `--format=json`** unless the user passed `--format` themselves. Means it consumes JSON from stdout — don't add stderr-merging there or you'll break the JSON parse (`filter_lint.go:65-71`).
- **`gotk tsc` merges stderr into stdout** (`filter_tsc.go:32`) so the regex parser sees both streams. This is *intentional* and contrary to the project-wide "stderr passes through verbatim" rule — read the comment-free code carefully before changing.
- **History file format is JSONL with field names `ts/cmd/raw/filtered/ms/exit`** (`track.go:21-28`). `gain.go` parses these — changing the schema breaks `gotk gain` for users with existing history.
- **Module is `gotk` (no domain prefix).** `go.mod` says `module gotk`, so `go install github.com/<you>/gotk@latest` requires editing the module path first. README's "once published" snippet flags this.

## Things to avoid

- **Don't refactor the flat layout** (`cmd/` / `internal/` split). It's a deliberate small-tool choice — see `docs/project/conventions.md` "Don't refactor".

- **Don't introduce hooks / shell integration / auto-rewrites.** `GOTK.md` "Non-goals for v0.1" calls this out: gotk is *instruction-driven* on purpose.
